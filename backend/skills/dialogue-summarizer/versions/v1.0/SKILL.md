# 对话总结与复盘技能

## 技能描述
这个技能用于对当前对话上下文进行智能总结和复盘，提取关键信息、识别行动项、生成结构化回顾。

## 使用场景
当用户说以下类型的话时触发：
- "总结一下我们的对话"
- "刚才我们讨论了什么？"
- "帮我复盘一下这次对话"
- "提取对话中的关键点"
- "生成会议纪要"
- "有哪些行动项需要跟进？"
- "对话回顾"

## 输入要求
技能需要访问当前对话的上下文信息。通常由 Agent 系统提供对话历史。

## 执行步骤

### 1. 收集对话上下文
- 从系统获取最近的对话历史（通常是最后10-20轮对话）
- 如果对话很长，可以分段处理

### 2. 分析对话结构
- 识别对话主题和子话题
- 标记重要决策点
- 识别问题和解决方案
- 提取关键数据和事实

## 技术实现建议

### 1. 上下文获取
- 依赖 Agent 系统提供的对话历史接口
- 如果系统不提供，可以请求用户提供对话摘要

### 2. 信息提取策略
- 使用关键词提取识别重要概念
- 通过问题模式识别（谁、什么、何时、哪里、为什么、如何）
- 识别决策语句（"决定"、"同意"、"选择"等）
- 提取数字和日期信息

### 3. 自然语言处理
- 可以使用简单的规则匹配
- 也可以集成更复杂的 NLP 模型（如果可用）
- 重点放在结构化信息提取上

**输入**: 3-5轮关于天气查询的对话

**输入**: 10轮关于编程问题的讨论
**预期输出**: 包含问题描述、解决方案、代码示例、后续步骤的总结

### 测试3：项目规划对话
**输入**: 15轮关于项目规划的对话
**预期输出**: 包含项目目标、任务分配、时间线、风险点的详细总结

## 注意事项

1. **隐私保护**: 不要总结包含敏感信息的对话
2. **准确性**: 确保总结准确反映对话内容
3. **客观性**: 保持中立客观的总结立场
4. **实用性**: 提供对用户真正有用的信息
5. **可读性**: 使用清晰易懂的语言和格式

## 技能优化建议

1. **个性化模板**: 允许用户自定义总结模板
2. **多语言支持**: 支持不同语言的对话总结
3. **情感分析**: 加入对话情感倾向分析
4. **主题聚类**: 自动聚类相关话题
5. **时间线可视化**: 生成对话时间线图
