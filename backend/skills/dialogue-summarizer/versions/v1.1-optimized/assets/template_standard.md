# 标准总结模板

## 适用场景
- 中等长度对话（6-15轮）
- 项目讨论和技术交流
- 正式会议记录
- 需要结构化总结的场景

## 模板结构

```
## 对话总结报告

### 📋 基本信息
- **对话时间**: {{start_time}} - {{end_time}}
- **对话时长**: {{duration}}
- **参与方**: {{participants}}
- **主要话题**: {{main_topics}}
- **总结模式**: 标准

### 🎯 关键讨论点
{{#each discussion_points}}
{{@index}}. **{{this.title}}**
   - **背景/问题**: {{this.context}}
   - **讨论内容**: {{this.discussion}}
   - **结论/决定**: {{this.conclusion}}
{{/each}}

### ✅ 已达成共识
{{#each consensus}}
- {{this}}
{{/each}}

### 📝 行动项与待办事项
| 事项 | 负责人 | 截止时间 | 状态 | 优先级 |
|------|--------|----------|------|--------|
{{#each action_items}}
| {{this.item}} | {{this.assignee}} | {{this.deadline}} | {{this.status}} | {{this.priority}} |
{{/each}}

### 💡 复盘建议
1. **做得好的方面**:
   {{#each strengths}}
   - {{this}}
   {{/each}}

2. **可以改进的方面**:
   {{#each improvements}}
   - {{this}}
   {{/each}}

3. **下次对话建议**:
   {{#each suggestions}}
   - {{this}}
   {{/each}}

### 📊 数据统计
- 总对话轮数: {{total_turns}}
- 用户发言次数: {{user_turns}}
- AI发言次数: {{ai_turns}}
- 问题解决率: {{resolution_rate}}%
- 行动项数量: {{action_count}}
```

## 字段说明

### 基本信息
- **start_time**: 对话开始时间
- **end_time**: 对话结束时间（或当前时间）
- **duration**: 对话时长（如：约15分钟）
- **participants**: 参与方（如：用户和AI助手）
- **main_topics**: 主要话题，1-3个

### 关键讨论点
- **title**: 讨论点标题
- **context**: 讨论的背景或问题
- **discussion**: 讨论的具体内容
- **conclusion**: 达成的结论或决定

### 已达成共识
- 明确达成一致的观点或决定
- 使用简洁的陈述句
- 按重要性排序

### 行动项表格
- **item**: 具体事项描述
- **assignee**: 负责人（用户/AI/待确定）
- **deadline**: 截止时间
- **status**: 状态（待处理/进行中/已完成）
- **priority**: 优先级（高/中/低）

### 复盘建议
- **strengths**: 对话中做得好的方面
- **improvements**: 可以改进的方面
- **suggestions**: 对下次对话的建议

### 数据统计
- **total_turns**: 对话总轮数
- **user_turns**: 用户发言次数
- **ai_turns**: AI发言次数
- **resolution_rate**: 问题解决率（已解决问题/总问题数）
- **action_count**: 识别出的行动项数量

## 生成规则

### 1. 讨论点提取规则
1. **问题识别**: 识别对话中的明确问题
2. **解决方案**: 对应问题的讨论和解决方案
3. **决策过程**: 如何达成最终决定
4. **重要转折**: 对话方向的变化点

### 2. 共识识别规则
- 双方明确表示同意的观点
- 重复确认的重要信息
- 共同认可的行动计划
- 无争议的技术方案

### 3. 行动项提取规则
1. **明确责任**: 有明确负责人的事项
2. **时间要求**: 有明确时间要求的事项
3. **具体行动**: 可执行的具体操作
4. **重要性**: 对对话目标有直接影响的事项

### 4. 复盘建议生成规则
- **strengths**: 基于有效沟通和问题解决
- **improvements**: 基于沟通障碍或信息缺失
- **suggestions**: 基于未完成事项或后续步骤

## 优化提示

### 内容组织
- 按重要性排序讨论点
- 将相关讨论点分组
- 使用一致的术语和表达

### 语言表达
- 使用客观中立的语言
- 避免主观判断和情感色彩
- 使用清晰准确的技术术语

### 格式规范
- 保持表格对齐
- 使用一致的标题风格
- 合理使用空白行分隔章节
