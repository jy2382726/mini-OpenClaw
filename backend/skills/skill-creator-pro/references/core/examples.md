# Examples

## Strong Pattern
A log-processing skill that:
- clearly names scanning, extraction, and archival branches
- moves parsing rules into references
- puts report templates into assets
- validates generated outputs

## Weak Pattern
A "data skill" that:
- claims to help with any data task
- explains what CSV and JSON are
- has no scripts even though it repeats the same transforms
- has no validation guidance

## Rewrite Example
Weak description:
`Help with data workflows.`

Stronger description:
`Clean, transform, and package repeated local data-processing workflows with reusable scripts, validation steps, and reference guidance. Use when asked to standardize recurring CSV, JSON, or tabular file processing tasks into a reusable skill.`

## Expansion Rule
Do not add new branches just because they are adjacent.
Add a branch only after a real usage pattern appears or a clear planned need exists.
