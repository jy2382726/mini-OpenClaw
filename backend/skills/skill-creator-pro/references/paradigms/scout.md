# Scout（环境侦察）

> 占比：6% — 高风险场景的安全保障范式

## 定义

**核心职责**：Inspect the environment before acting.

**适用场景**：
- 未知仓库探索（第一次接触的代码库）
- 高风险编辑（生产环境修改、不可逆操作）
- 调查密集型工作（Bug 定位、性能分析、安全审计）
- 环境验证（部署前检查、依赖兼容性确认）

Scout 的本质是"先看再做"的执行纪律。它强制 Agent 在采取任何行动之前，先通过侦察收集环境信息，基于证据而非假设做出决策。

---

## SKILL.md 结构指导

### 保留在 SKILL.md 中的内容

- **Recon-First Workflow**：侦察优先的执行流程
- **Safety Rules**：安全规则（什么情况下禁止行动）
- **First Inspection Commands**：首次侦察时执行的命令列表
- **Stop Conditions**：何时停止侦察、开始行动

### 推入 `references/` 目录的内容

- 环境特定详情（不同类型仓库的侦察清单）
- 已知问题模式（常见环境问题的特征和解法）

```
skill-name/
├── SKILL.md          # 侦察流程 + 安全规则 + 首次检查 + 停止条件
└── references/
    ├── checklists/   # 各环境类型的侦察清单
    └── patterns/     # 已知问题模式库
```

## 关键设计原则

### 1. 侦察优先（Recon-First）

任何操作之前，必须先执行侦察步骤。侦察的目的是建立环境的心智模型：
- 目录结构是什么样的
- 关键配置文件在哪里
- 当前状态是什么（git status、运行状态、版本信息）

### 2. 证据驱动（Evidence-Driven）

所有决策必须基于侦察收集到的证据，而非假设或经验。关键规则：
- 不要假设文件存在，先检查
- 不要假设版本兼容，先验证
- 不要假设环境配置正确，先确认

### 3. 反猜测规则（Anti-Guessing）

当侦察结果不足以支撑决策时，必须：
- 执行额外侦察（而非猜测）
- 或向用户请求更多信息（而非假设）
- 禁止基于"通常是这样"的经验进行判断

### 4. 停止条件（Stop Conditions）

侦察不能无限进行。SKILL.md 必须定义明确的停止条件：
- 时间限制：侦察阶段不超过 N 步
- 充分条件：收集到 X 类信息即可开始行动
- 升级条件：侦察 N 步后仍无法确定，请求人工介入

## 反模式警告

### 1. 没有侦察就行动

**症状**：Agent 收到任务后直接开始修改文件，没有先检查环境。
**后果**：可能修改错误的文件、使用错误的工具版本、破坏现有功能。
**修复**：在 SKILL.md 开头强制定义 Recon Phase，列出必须执行的首次检查命令。

### 2. 基于假设判断

**症状**：Agent 基于"一般来说"、"通常是"做出决策，而非基于实际侦察结果。
**后果**：在非典型环境中产生错误判断。
**修复**：添加 Anti-Guessing 规则，要求每个决策引用具体的侦察证据。

### 3. 无限侦察

**症状**：Agent 不断收集信息但永远不开始行动，陷入分析瘫痪。
**后果**：任务无法完成，资源浪费。
**修复**：设定明确的停止条件和时间预算。侦察阶段不应超过总工作量的 20-30%。

## 典型代表

### systematic-debugging（系统化调试）

- Recon Phase: 收集错误日志、环境信息、最近变更
- 证据驱动：根据日志定位问题，而非猜测原因
- 停止条件：找到可复现的最小案例后开始修复
- Safety Rules: 修复前先创建备份/分支
- 典型 Scout：先观察再行动 + 证据链

### verification-before-completion（完成前验证）

- 在声称任务完成之前，执行全面验证
- 侦察清单：构建通过？测试通过？功能可用？
- 反猜测：不假设"应该没问题"，实际验证每个声明
- 停止条件：所有验证项通过或已知问题已记录

### oscar-research（Oscar 研究助手）

- 研究任务前先侦察已有知识和资源
- 多轮侦察：先广度搜索 → 再深度挖掘
- 证据收集：每个结论必须附带来源链接
- 停止条件：覆盖足够的信息维度后输出报告

## 与相邻范式的区别

### Scout vs Navigator

| 维度 | Scout | Navigator |
|------|-------|-----------|
| 目标 | 侦察**未知**环境 | 路由到**已知**信息 |
| 信息状态 | 信息需要被发现 | 信息已经存在 |
| 动作模式 | 探索 + 发现 | 查找 + 导航 |
| 确定性 | 低（结果未知） | 高（路由预定义） |

### Scout vs Partner

| 维度 | Scout | Partner |
|------|-------|---------|
| 检查对象 | **环境**状态 | **用户**意图 |
| 信息来源 | 文件系统、命令输出 | 用户对话 |
| 目的 | 建立环境心智模型 | 确认需求和方向 |
| 自主性 | Agent 自主侦察 | 依赖用户参与 |
